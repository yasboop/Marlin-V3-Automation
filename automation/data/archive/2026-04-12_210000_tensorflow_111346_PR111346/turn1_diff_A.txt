+# TensorFlow / XLA Memory Space Assignment
+
+## Overview
+
+This is the TensorFlow repository. The area of focus is XLA's Memory Space Assignment (MSA) module located at `third_party/xla/xla/service/memory_space_assignment/`.
+
+MSA is a compiler pass that decides whether HLO buffers should live in default memory (HBM) or faster alternate memory (VMEM/SRAM). It handles prefetching buffers from default to alternate memory, evicting them back, and scheduling these async copies to overlap with computation.
+
+## Key Source Files
+
+- `algorithm.cc` / `algorithm.h` — Core MSA algorithm (`MsaAlgorithm` class). Contains `AllocateAllocationValues`, `AllocateSegment`, `CreateAllocationRequest`, and all the logic for deciding memory placement.
+- `allocation.cc` / `allocation.h` — Allocation type hierarchy. Base class `Allocation` with subclasses: `PinnedAllocation`, `CopyAllocation`, `SlicedCopyAllocation`, `MirroredAllocation`, `ParentAllocation`, `WindowPrefetchedAllocation`, `ScopedAllocation`, `ReservedAllocation`.
+- `memory_space_assignment.cc` / `memory_space_assignment.h` — Top-level pass that runs the algorithm, processes allocations, and verifies results.
+- `allocation_value.cc` / `allocation_value.h` — `AllocationValue` represents an HLO value that needs memory assignment, with its uses and defining position.
+- `options.h` — Configuration options for the algorithm.
+
+## Build System
+
+TensorFlow uses Bazel. BUILD files define targets and dependencies.
+
+```bash
+# Build MSA library
+bazel build //third_party/xla/xla/service/memory_space_assignment:memory_space_assignment
+
+# Run MSA tests
+bazel test //third_party/xla/xla/service/memory_space_assignment:memory_space_assignment_test
+```
+
+## Code Conventions
+
+- C++17 with Google style
+- Uses `absl::Status` / `absl::StatusOr` for error handling
+- HLO instructions use opcodes like `HloOpcode::kConditional`, `HloOpcode::kWhile`
+- Memory spaces: `MemorySpace::kDefault` (HBM) and `MemorySpace::kAlternate` (VMEM)
+- Allocations track start/end times in the HLO schedule
+- `VLOG(N)` for debug logging at various verbosity levels
+- Tests use `ParseAndReturnVerifiedModule` to create HLO modules from string, then `AssignMemorySpace` to run MSA
+
+## Architecture Notes
+
+- The algorithm processes allocation values in priority order (determined by `MsaBufferIntervalCompare`)
+- For each value, it iterates through uses and calls `AllocateSegment` to find placement
+- `MirroredAllocation` currently only handles while-loop eviction scenarios in default memory
+- Conditional operations have special handling: operands passed into conditionals may need eviction if the branches modify memory
+- `AliasedOffset` tracks which allocations share the same physical memory chunk
+- `CreateOrAddToAliasedOffset` manages the offset map and asserts no duplicates

+bool MsaAlgorithm::AllConditionalBranchesCanUseAlternateMem(
+    const AllocationValue& value, const AllocationValue::Use& use) const {
+  CHECK_EQ(use.hlo_use.instruction->opcode(), HloOpcode::kConditional);
+  const auto& instruction_schedule = hlo_live_range_.instruction_schedule();
+  int64_t conditional_time = instruction_schedule.at(use.hlo_use.instruction);
+
+  // Collect all uses of this value that target the same conditional.
+  bool all_branches_ok = true;
+  bool found_any_branch = false;
+  for (const AllocationValue::Use& other_use : value.uses()) {
+    if (other_use.hlo_use.instruction != use.hlo_use.instruction) {
+      continue;
+    }
+    // Operand 0 is the predicate, not passed into branch computations.
+    if (other_use.hlo_use.operand_number == 0) {
+      continue;
+    }
+    found_any_branch = true;
+    HloComputation* called_computation =
+        use.hlo_use.instruction->called_computations().at(
+            other_use.hlo_use.operand_number - 1);
+    const HloInstruction* parameter_instruction =
+        called_computation->parameter_instruction(0);
+    HloValue* parameter_value =
+        &alias_analysis_.dataflow_analysis().GetUniqueValueAt(
+            parameter_instruction, other_use.hlo_use.operand_index);
+    int64_t parameter_time = instruction_schedule.at(parameter_instruction);
+    int64_t min_use_time = conditional_time;
+    for (const HloUse& parameter_use : parameter_value->GetUses()) {
+      if (parameter_use.instruction->parent() == called_computation &&
+          parameter_use.instruction->opcode() !=
+              HloOpcode::kGetTupleElement &&
+          parameter_use.instruction->opcode() != HloOpcode::kTuple &&
+          parameter_use.instruction->opcode() != HloOpcode::kBitcast) {
+        min_use_time = std::min(min_use_time,
+                                instruction_schedule.at(
+                                    parameter_use.instruction));
+      }
+    }
+    if (!options_.prefetch_interval_picker->CanAllocateInAlternateMemoryNoCopy(
+            parameter_value->shape(), parameter_time, min_use_time)) {
+      VLOG(4) << "Conditional branch " << called_computation->name()
+              << " cannot use alternate memory (parameter_time="
+              << parameter_time << ", min_use_time=" << min_use_time << ")";
+      all_branches_ok = false;
+      break;
+    }
+  }
+  return found_any_branch && all_branches_ok;
+}

+void MsaAlgorithm::MaybeCreateMirroredAllocationForConditionalUse(
+    const AllocationValue& allocation_value, const AllocationValue::Use& use,
+    int64_t use_time, absl::Span<AllocationValue> allocation_values,
+    absl::flat_hash_map<const HloComputation*, AliasedOffset*>&
+        preferred_offset_for_computation) {
+  const HloUse& hlo_use = use.hlo_use;
+
+  if (hlo_use.instruction->opcode() != HloOpcode::kConditional) {
+    return;
+  }
+
+  if (!options_.enable_conditional_redundant_eviction_elimination) {
+    return;
+  }
+
+  Allocation* aliased_allocation =
+      GetLiveAllocationAt(*allocation_value.allocation_sequence(), use_time);
+  if (aliased_allocation->memory_space() != MemorySpace::kAlternate) {
+    return;
+  }
+
+  if (!AllConditionalBranchesCanUseAlternateMem(allocation_value, use)) {
+    VLOG(3) << "Not all conditional branches can use alternate memory for "
+            << allocation_value.ToShortString() << " at use "
+            << hlo_use.ToString();
+    return;
+  }
+
+  VLOG(3) << "Creating mirrored allocations for conditional branches: "
+          << allocation_value.ToShortString() << " at use "
+          << hlo_use.ToString();
+
+  const auto& instruction_schedule = hlo_live_range_.instruction_schedule();
+  AliasedOffset* aliased_offset = GetAliasedOffset(*aliased_allocation);
+
+  // Create a mirrored allocation in alternate memory for each branch
+  // computation's parameter.
+  for (const AllocationValue::Use& other_use : allocation_value.uses()) {
+    if (other_use.hlo_use.instruction != hlo_use.instruction) {
+      continue;
+    }
+    // Operand 0 is the predicate, not passed into branch computations.
+    if (other_use.hlo_use.operand_number == 0) {
+      continue;
+    }
+    HloComputation* called_computation =
+        hlo_use.instruction->called_computations().at(
+            other_use.hlo_use.operand_number - 1);
+
+    // Find the AllocationValue for the parameter in this branch computation.
+    auto branch_alloc_value_it =
+        absl::c_find_if(allocation_values, [&](const AllocationValue& value) {
+          return value.computation() == called_computation &&
+                 value.defining_instruction()->opcode() ==
+                     HloOpcode::kParameter;
+        });
+    if (branch_alloc_value_it == allocation_values.end()) {
+      VLOG(3) << "Could not find branch allocation value for computation "
+              << called_computation->name();
+      continue;
+    }
+
+    int64_t parameter_time = instruction_schedule.at(
+        branch_alloc_value_it->defining_instruction());
+
+    branch_alloc_value_it->mutable_allocation_sequence()->push_back(
+        std::make_unique<MirroredAllocation>(*aliased_allocation,
+                                             parameter_time,
+                                             MemorySpace::kAlternate));
+    VLOG(3) << "Created mirrored allocation in alternate memory for "
+            << called_computation->name() << ": "
+            << branch_alloc_value_it->allocation_sequence()
+                   ->back()
+                   ->ToString();
+
+    CreateOrAddToAliasedOffset(
+        *branch_alloc_value_it->allocation_sequence()->back(), aliased_offset);
+
+    preferred_offset_for_computation[called_computation] = aliased_offset;
+  }
