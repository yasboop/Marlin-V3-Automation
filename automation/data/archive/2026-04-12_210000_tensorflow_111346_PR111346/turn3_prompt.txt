Looking at MaybeCreateMirroredAllocationForConditionalUse in algorithm.cc ,the function is not checking for async conversion candidates inside the branch computations right now. If a branch has an instruction that is basically an async conversion candidate the mirrored allocation would just end up as an alt to alt copy which is not supported. For those cases the whole thing needs to fall back to the default eviction path instead of creating the mirrored allocation.

For VerifyAllocations in memory_space_assignment.cc ,the check is using dynamic_cast to figure out if something is a MirroredAllocation. That approach is not great when the codebase already has a proper allocation hierarchy with virtual methods sitting there. A virtual method on Allocation would be cleaner and actually consistent with how the rest of the hierarchy works
